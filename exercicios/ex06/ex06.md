### 6 - What are the problems you will face when trying to match ORB with SIFT features?

O principal problema que podemos encontrar é que os dois algoritmos possuem descritores diferentes. 
O ORB vai criar o descritor binário, enquanto o SIFT vai criar descritor do tipo float. Por um lado, o descritor binário permite que as operações de matching sejam bem mais rápidas, uma vez que utiliza a distância de Hamming para comparar, porém perde qualidade dos dados, enquanto o descritor do tipo float, apesar de ser mais pesado computacionalmente por utilizar a distância Euclidiana, garante informações mais detalhadas sobre a imagem (gradiente dos keypoints). 
Dessa forma, os tamanhos dos descritores vão ser diferentes, os tipos de dados são diferentes e a comparação realizada é diferente. Logo, esse é o problema que enfrentaremos, os descritores precisam ser "iguais", e, para fazer isso, acredito que o modo mais fácil seja utilizando o descritor BRIEF no próprio SIFT (então ele só faz a extração dos keypoints), mas o tamanho do vetor tem que ser igual.

**OBS:** Talvez não faça sentido comparar SIFT e ORB mesmo utilizando o mesmo descritor, já que o ORB usa BRIEF, pois são propostas diferentes dos algoritmos.
